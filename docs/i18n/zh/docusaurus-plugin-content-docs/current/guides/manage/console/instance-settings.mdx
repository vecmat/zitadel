title: 实例设置
---

实例设置可作为默认设置或后备设置用于您的组织设置。 在大多数情况下，您只需设置实例设置，用于在组织本身不需要特定行为或者您只有一个组织的情况。

要访问实例设置。 使用 `{instanceDomain}/ui/console/setting` 或点击页面 **右上角的实例按钮** 然后导航到导航中的设置。

<img
  src="/img/guides/console/instancebutton.png"
  alt="实例按钮"
  width="450px"
 />

当您配置您的实例时，您可以设置以下内容：

- **常规**: 界面的默认语言
- [**通知提供者和 SMTP**](#notification-providers-and-smtp): 电子邮件服务器设置, 所以初始化、验证和其他邮件都是从您自己的域发送的。 SMS 支持 Twilio 作为通知提供者。
- [**登录行为和访问**](#login-behaviour-and-access): 多重身份验证选项和执行， 定义是否允许使用无密码身份验证方法，设置登录生命期和高级登录界面的操作符。
- [**身份提供者**](#identity-providers)：定义可供所有组织使用的国内流离失所者
- [**密码复杂程度**](#password-complexity): 密码的要求 ex. 符号，数字，最小长度和更多。
- [**锁定**](#lockout): 设置用户可以尝试输入密码的最大尝试次数 当数字超出时，用户会被锁定，必须解锁。
- [**域设置**](#domain-settings): 用户是否使用他们的电子邮件或生成的用户名登录。 其他校验，SMTP设置
- [**品牌**](#branding): 登录界面外观。
- [**消息文本**](#message-texts)：电子邮件的文本和国际化
- [**登录界面文本**](#login-interface-texts): 登录界面的文本和国际化
- [**隐私政策**](#privacy-policy-and-tos): 链接到您自己的服务条款和隐私政策规则。 链接到帮助页面。
- [**OIDC 令牌生命期和过期**](#oidc-token-lifetimes-and-expiration): 令牌有效期和过期设置。
- [**秘密外观**](#secret-appearance): 用于验证等邮件生成的代码和秘密的外观

## 品牌化

我们建议先设置您的品牌和SMTP设置，因为这将使您的客户感到舒适，您有一个熟悉的用户界面可以登录并接收来自您的域名和邮件地址的通知。

![私有标签](/img/console_private_labeling.png)

在品牌设置中，您可以为登录界面上传您的徽标。 为按钮、背景、链接设置您自己的颜色，并在多个符号之间进行选择。 您不必是专家，因为这些设置都可以在没有任何CSS知识的情况下设置。

| 设置   | 描述                                                                                                                 |
| ---- | ------------------------------------------------------------------------------------------------------------------ |
| 徽标   | 上传您的徽标来显示灯光和暗色设计。 这主要用于登录界面。                                                                                       |
| 图标   | 上传您的图标以显示灯光和暗色设计。 图标用于较小的组件。 例如，在顶部左侧的控制台中作为主页按钮。                                                                  |
| 颜色   | 您可以设置四种不同的颜色来设计您的登录页面和电子邮件。 (背景，初级，警告和字体颜色)                                                                        |
| Font | 上传您的自定义字体                                                                                                          |
| 高级行为 | **隐藏 Loginname 后缀**: 如果启用，您的 loginname 后缀(Domain) 将不会显示在登录页面中。 **禁用 Watermark**：如果您禁用水印，您将不会在登录页面看到"由ZITADEL提供动力"。 |

请确保您在完成配置后点击"应用配置"按钮。 这将确保您的设计对您的客户可见。

在您的实例上应用的品牌设置作为您所有组织的默认值。 如果您需要自定义品牌到某个组织，请查看 [organisation settiong](./organizations#branding) 下我们的指南。

## 通知提供者和 SMTP

在通知设置中，您可以配置您的 SMTP 服务器设置和短信提供商。 目前，Twilio 可用作为短信提供者。

### SMTP

在每个实例中，我们都配置我们默认的 SMTP 提供商。 为了确保您只从您拥有的域名发送一些E-Mail。 您需要在您的实例上添加一个自定义域名。 前往ZITADEL [客户端门户](https://zitadel.cloud) 配置一个自定义域。

要配置您的自定义SMTP，请填写以下字段：

- 发件人电子邮件地址
- 发件人名称
- 启用或禁用传输图层安全(TLS)
- 主机
- 用户
- SMTP 密码

<img src="/img/guides/console/smtp.png" alt="SMTP" width="400px" />

### 短信

没有默认提供商被配置为向您的用户发送一些短信。 如果您想要验证您的用户的电话号码，请通过添加您的密码来添加您的 twilio 配置 令牌和发件人号码。

## 登录行为和访问

登录策略定义了登录过程应该如何看起来以及用户必须验证的验证选项。

| 设置          | 描述                                                                |
| ----------- | ----------------------------------------------------------------- |
| 允许注册        | 在登录界面启用自注册的可能性，这将使用户名密码注册以及在配置的外部身份提供商注册                          |
| 用户名密码已允许    | 使用用户名和密码登录的可能性。 如果禁用此功能，只允许使用外部身份提供者登录                            |
| 允许外部国内流离失所者 | 使用外部身份登录的可能性(e)。 Google、Microsoft、Apple 等。如果您想允许外部身份提供商将其添加到提供商列表 |
| 强制MFA       | 强制用户注册并使用多元素身份验证，确保您已经添加了您想要允许的 MFA 方法。                           |
| 无密码         | 选择是否允许无密码登录                                                       |

### 无密码

无密码身份验证意味着用户不需要输入密码才能登录。 在我们的情况下，用户必须输入他的登录名，并作为下一步通过注册设备或令牌证明身份的。 有两种不同的类型，一种取决于设备(例如指纹、脸识别、WindowsHello)，另一种是独立的(例如Yubikey、Solokey)。

### 多因子

在多因素部分中，您可以配置允许的多因素类型。 无密码工作需要启用 U2F (Universal Second Factor) 与 PIN。 目前没有其他选择。 多因子：

- U2F (通用的第二因素) 与 PIN, 例如Facebook, WindowsHello, 指纹, Hardwaretokens such Yubikey

二级因素：

- OTP (一次密码), 身份验证器应用程序, 例如Google/Microsoft 身份验证器, 作者等。
- U2F (通用第二因素)，例如Facebook, WindowsHello, 指纹, Hardwaretokens such Yubikey

### 登录寿命

配置不同的生命周期检查登录过程：

- **密码检查生命期** 指定用户在登录过程中必须重新输入他的密码
- **外部登录检查生命期** 指定用户在登录过程中被重定向到IDP
- **多因子Init Lifetime** 指定了在登录过程中提醒用户设置两个因素/多因素的时间段之后(值 0 将停用提示)
- **第二个因素检查生命期** 指定用户在登录过程中必须重新验证两个因素的后期时间
- **外部登录检查生命期** 指定用户在登录过程中必须重新验证多因素的时间间隔后

关于如何为身份经纪活动配置新的身份提供商的详细指南，请参阅我们的指南： [身份经纪](../../../guides/integrate/identity-brokering)

## 锁定

定义一个账户应该被锁定的时间。

以下设置是可用的：

- 最大密码尝试次数：当用户已达到密码最大尝试时，账户将被锁定， 如果设置为 0，锁定将不会触发。

如果帐户被锁定，管理员必须在ZITADEL控制台中解锁

## 域设置

在域策略中，您有两种不同的设置。 一个是"user_login_must_be_domain"，通过设置这个组织内的所有用户将被该组织的域所覆盖。

第二个是"validate_org_domains"，如果设置为一个组织上所有创建的域名都必须按每个acme 挑战进行验证。 更多关于如何验证域 [](../../../guides/manage/console/organizations#domain-verification-and-primary-domain)。 如果设置为 false， 所有注册的域将自动创建，用户将能够使用该域登录。

### 使用电子邮件作为用户名

要能够使用电子邮件作为用户名，您必须在域设置中禁用属性“用户登录名必须包含 orgdomain”。 这意味着您的所有用户将不会被组织的域设置为后缀，您可以输入电子邮件作为用户名。 所有用户名将在您的实例中全局唯一。

您可以将此属性设置在您的整个ZITADEL实例上，或者只设置在一些特定的高度。

## 隐私政策和 TOS

通过这个设置，您可以配置您的隐私政策、服务条款和帮助链接。 注册时，每个用户必须接受这些政策。

您的组织也可以推翻此政策。

当聚焦于输入字段时，您可以看到语言属性，然后可以整合到您的链接。

例如： `https://demo.com/tos-{{.Lang}}`
